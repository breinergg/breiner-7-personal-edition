PreviDocs — Sistema Especializado de Gestión de Historias Clínicas

Descripción General

PreviDocs es un sistema integral de gestión de historias clínicas diseñado específicamente para instituciones de salud mental y psicología clínica. Esta aplicación web cliente-servidor optimiza el registro, seguimiento y consulta de información médica, abarcando desde los datos de pacientes hasta las sesiones terapéuticas y las evoluciones clínicas o familiares. Además, agiliza los procesos administrativos e informativos mediante la generación automatizada de reportes detallados en formato PDF.

Características Principales

  • Gestión de Pacientes e Historias Clínicas: Sistema avanzado de búsqueda inteligente que integra bases de datos locales y externas (como sistemas de enfermería), permitiendo la creación de historias clínicas mediante formularios exhaustivos que documentan diagnósticos, antecedentes y planes de tratamiento.
  • Seguimiento de Sesiones y Evoluciones: Módulo dedicado para registrar sesiones terapéuticas con numeración secuencial automática, así como el seguimiento de la evolución clínica de cada paciente por sesión. Incluye autocompletado dinámico de diagnósticos y profesionales consultando un catálogo externo.
  • Generación de Reportes y Dashboard: Capacidad para generar documentos PDF multipágina completos que incluyen desde el diagrama familiar hasta la impresión diagnóstica y la firma digital. El sistema también cuenta con un panel de control interactivo (Dashboard) con animaciones de métricas en tiempo real sobre pacientes, historias y sesiones totales.
  • Seguridad y Control de Acceso: Implementación de un esquema de seguridad mediante tokens JWT con expiración. Cuenta con protección de rutas en el cliente y control de acceso basado en roles específicos (Administrador, Médico, Facturador) para garantizar la confidencialidad de la información clínica.

Arquitectura y Tecnologías (Stack Tecnológico)

El proyecto fue construido bajo una arquitectura robusta de dos capas, garantizando la escalabilidad, separación de responsabilidades y el bajo acoplamiento mediante inyección de dependencias.

Frontend (Client-Side)

  • Framework: Desarrollado como una Single Page Application (SPA) con Angular 19.2 utilizando Standalone Components.
  • Arquitectura y Estado: Diseño modular jerárquico basado en páginas/componentes, con gestión de estado reactiva apoyada en RxJS para la sesión de usuario.
  • Seguridad: Uso de guards funcionales para proteger el acceso a rutas privadas y un interceptor HTTP que inyecta automáticamente los tokens JWT en cada petición.
  • UI/UX: Interfaz fluida con formularios reactivos, validaciones, paginación del lado del cliente, filtros dinámicos y autocompletado optimizado con debounce para no saturar el servidor.

Backend (Server-Side)

  • Framework: API REST de alto rendimiento construida con ASP.NET Core 8.0 y C# 12.
  • Patrones de Diseño: Arquitectura fuertemente tipada estructurada en capas (Controladores, Servicios y Repositorios), complementada con el uso extensivo de DTOs (mapeados mediante Mapster) para gestionar eficientemente las transferencias de datos.
  • Base de Datos y ORM: Persistencia implementada sobre una base de datos relacional Firebird, utilizando Dapper como micro-ORM para ejecutar consultas SQL directas y de gran velocidad.
